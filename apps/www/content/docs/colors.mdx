## Overview

Pixel UI uses a carefully curated NES 8-bit inspired color palette that provides excellent contrast and maintains the retro aesthetic. All colors automatically adapt for dark mode where appropriate.

## Usage in Components

### Component Variants

All interactive components support these variant colors through the `variant` prop:

```tsx
import { Button } from '@joacod/pixel-ui'

// Available variants
<Button variant="base">Base</Button>
<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="accent">Accent</Button>
<Button variant="ghost">Ghost</Button>
<Button variant="error">Error</Button>
<Button variant="success">Success</Button>
<Button variant="warning">Warning</Button>
```

**Variant descriptions:**

- **base**: Adapts between black (light mode) and white (dark mode)
- **primary**: Main brand color
- **secondary**: Alternative accent
- **accent**: Highlight color
- **ghost**: Subtle gray variant
- **error**: Error states
- **success**: Success states
- **warning**: Warning states

## Dark Mode

Pixel UI components automatically adapt to dark mode with no configuration needed. When your app switches to dark mode:

- **Base colors adapt**: Black backgrounds become gray-dark, borders become white
- **Semantic colors stay consistent**: Primary, secondary, accent, etc. remain the same for brand recognition
- **Components handle everything**: All dark mode styling is built into the components

You don't need to apply any dark mode classes - the components respond to your system's `prefers-color-scheme` automatically.
