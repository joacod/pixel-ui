## Usage in Components

### Component Variants

All interactive components support these variant colors:

- **base**: Adapts between black (light mode) and white (dark mode)
- **primary**: Main brand color (#3337FE)
- **secondary**: Alternative accent (#F15BFE)
- **accent**: Highlight color (#51DF21)
- **ghost**: Subtle gray variant
- **error**: Error states (#FE7269)
- **success**: Success states (#51DF21)
- **warning**: Warning states (#ADB600)

### Tailwind CSS Classes

All colors are available as Tailwind utilities:

```tsx
// Background colors
<div className="bg-nes-primary" />
<div className="bg-nes-secondary" />

// Text colors
<p className="text-nes-error" />
<p className="text-nes-success" />

// Border colors
<div className="border-nes-accent" />

// Dark mode variants
<div className="bg-nes-black dark:bg-nes-white" />
<p className="text-nes-black dark:text-nes-white" />
```

## Dark Mode

Pixel UI components automatically adapt to dark mode. The base colors (black/white) swap, while semantic colors (primary, secondary, etc.) remain consistent for brand recognition.

### Dark Mode Color Mapping

- `nes-black` → Backgrounds in dark mode use `nes-gray-dark`
- `nes-white` → Text/borders in dark mode
- Semantic colors (primary, secondary, etc.) stay the same
- Borders and shadows automatically adapt for visibility
