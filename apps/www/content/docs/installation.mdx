## Installation

Install pixel-ui and its peer dependencies using your preferred package manager:

```package-install
@joacod/pixel-ui
```

## Peer Dependencies

pixel-ui requires the following peer dependencies:

```package-install
react react-dom tailwindcss @base-ui-components/react
```

## Setup

pixel-ui uses **Tailwind CSS v4's CSS-first configuration** approach. No JavaScript config files needed!

### 1. Configure PostCSS

Create or update `postcss.config.js` in your project root:

```js
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

### 2. Import Pixel UI Styles

In your app's main CSS file (e.g., `app/globals.css` or `src/index.css`):

```css
@import '@joacod/pixel-ui/components';
```

That's it! This single import includes everything you need:

- ✅ **Design tokens** - NES-inspired 8-bit color palette, 8px grid spacing, pixel fonts
- ✅ **Tailwind CSS** - Full utility classes for layout and styling
- ✅ **Base utilities** - Pixel-art helpers like `.pixel-border`, `.pixel-render`, `.pixel-text`
- ✅ **Component styles** - Pre-built styles for Button, Input, Checkbox, and all other components

<Callout type="info">
  All styles are pre-generated at build time, so you don't need to configure
  Tailwind to scan `node_modules` or add any `@source` directives. Just one
  import and you're ready to go!
</Callout>

## Usage

You can now import and use pixel-ui components:

```tsx
import { Button } from '@joacod/pixel-ui'

export default function App() {
  return (
    <Button variant="primary" size="md">
      Click me!
    </Button>
  )
}
```

## TypeScript Support

pixel-ui is written in TypeScript and includes type definitions out of the box. No additional setup is required.
